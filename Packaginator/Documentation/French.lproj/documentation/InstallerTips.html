Astuces pour le Programme d'installation

Mac OS X 10.1.5
  • Metapackages et packages en divers formats

    Si un metapackage au format 10.1 contient un package au format 10.1 et 10.2, le Programme d'installation ne pourra pas lancer l'installation.

    Suggestion :Supportez les formats 10.1 et 10.2 dans vos packages et metapackages si vous devez être compatible avec Mac OS X 10.1.

Mac OS X 10.2.x
  • Packages avec des binaires universels

    Du fait d'un bug dans le Programme d'installation, quand vous installez un package contenant des binaires universels sur Mac OS X 10.2.x et que ce package avait déjà été installé précédement, l'installation va planter.

    Suggestion :Effacer le reçu du package dans le script postflight si l'installation se déroule sous Mac OS X 10 < 10.3.

Mac OS X 10.3.x
  • Metapackage et package délocalisable

    Si un metapackage contient un package dont l'emplacement d'installation par défaut est délocalisable, l'emplacement choisi par l'utilisateur dans le Programme d'installation ne sera pas pris en compte.

    Suggestion :Aucune. Il s'agit d'un bug dans le Programme d'installation.

    Mac OS XStatus
    <= 10.2.8Non concerné
    10.3 - 10.3.3Concerné
    10.3.4Non concerné

  • Package et Autorisation Administrateur

    Il n'est pas recommandé de demander l'Autorisation Administrateur sous Mac OS X 10.3 si un package est délocalisable. Il existe un bug qui peut conduire à des permissions incorrectes pour certains composants de la destination d'installation par défaut.

    Suggestion :Aucune. Il s'agit d'un bug dans le Programme d'installation. Essayez d'éviter d'utiliser l'Autorisation Administrateur et demandez une Autorisation root à la place.

    Mac OS XStatus
    <= 10.2.8Non concerné
    = 10.3.xConcerné
